# automation-pom-cucumber
Reto de automatización

Este proyecto es un framework de automatización de pruebas funcionales desarrollado en Java, utilizando Selenium WebDriver, Serenity BDD y Cucumber, bajo el patrón de diseño Page Object Model (POM).

Su objetivo es automatizar pruebas end-to-end sobre aplicaciones web, garantizando pruebas mantenibles, escalables y fáciles de entender.
Los escenarios de prueba se definen en Gherkin, permitiendo una comunicación clara entre perfiles técnicos y no técnicos.

La arquitectura del proyecto separa de forma clara la lógica de las pruebas, la interacción con la interfaz de usuario y las definiciones de pasos, centralizando comportamientos comunes como esperas, validaciones e interacciones para promover la reutilización de código y la estabilidad de las ejecuciones.

El framework está preparado para el manejo de datos dinámicos, sincronización robusta y ejecución tanto en entornos locales como de integración continua.
